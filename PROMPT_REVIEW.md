## 프롬프트 작성 원칙

AI에게 코드를 요청할 때는 다음 원칙을 지킨다.

1. 한 번에 하나의 작은 기능만 요청한다.
2. 현재 단계의 목표를 먼저 설명한다.
3. 아직 구현하지 않을 기능도 명확히 제한한다.
4. C++ 초보자가 이해할 수 있는 수준을 요구한다.
5. 코드뿐 아니라 실행 흐름 설명도 요구한다.
6. 컴파일과 실행 방법을 함께 요구한다.
7. 정상 동작을 확인할 수 있는 테스트 방법을 요구한다.
8. 발생 가능한 오류와 주의점을 요구한다.
9. 코드의 한계와 다음 개선 방향을 요구한다.
10. 학생이 직접 설명할 수 있는 코드인지 확인하도록 요구한다.

## 단계별 프롬프트 생성 기준

새로운 개발 단계의 프롬프트를 만들 때는 아래 항목을 반드시 포함한다.

### 1. 현재 단계의 목적

이번 단계에서 무엇을 배우고 무엇을 구현할지 명확히 쓴다.

예시:

```text
이번 단계의 목표는 C++ 콘솔 프로그램에서 사용자 입력을 반복해서 받고,
/exit 명령어로 프로그램을 종료하는 기본 흐름을 이해하는 것이다.
```

### 2. 구현할 최소 기능

이번 단계에서 반드시 구현할 기능만 짧게 나열한다.

예시:

```text
1. 사용자 입력을 반복해서 받는다.
2. 입력 내용을 화면에 다시 출력한다.
3. /exit을 입력하면 종료한다.
```

### 3. 구현하지 않을 기능

아직 배우지 않았거나 현재 단계에서 필요 없는 기능은 명확히 제외한다.

예시:

```text
아직 클래스, 파일 저장, 대화 기록 저장, 실제 AI 모델 연동, 네트워크 통신은 넣지 말아줘.
```

### 4. 코드 수준 제한

학습 단계에 맞는 문법만 사용하도록 요구한다.

예시:

```text
C++ 초보자가 이해할 수 있도록 iostream, string, while, if 정도의 기본 문법만 사용해줘.
```

### 5. 설명 요구

코드 아래에 다음 설명을 요구한다.

1. 프로그램 실행 흐름
2. 주요 변수의 역할
3. 반복문과 조건문의 역할
4. 사용자가 입력한 값이 어떻게 처리되는지
5. 종료 조건이 어떻게 동작하는지

### 6. 검증 요구

학생이 직접 실행해 확인할 수 있는 검증 방법을 요구한다.

예시:

```text
정상 동작을 확인할 수 있는 테스트 입력과 예상 출력을 알려줘.
```

## 프롬프트 검토 항목

작성된 프롬프트는 아래 기준으로 검토한다.

| 번호 | 검토 항목 | 확인 질문 |
| --- | --- | --- |
| 1 | 목표 명확성 | 이번 단계에서 무엇을 배우는지 분명한가? |
| 2 | 기능 범위 | 한 번에 하나의 작은 기능만 요청하는가? |
| 3 | 실행 가능성 | 학생이 코드를 실행해서 결과를 확인할 수 있는가? |
| 4 | 수준 적합성 | C++ 초보자가 이해할 수 있는 문법 수준인가? |
| 5 | 제한 조건 | 아직 구현하지 않을 기능을 명확히 제외했는가? |
| 6 | 코드 설명 | 코드 구조와 실행 흐름 설명을 요구하는가? |
| 7 | 실행 방법 | 컴파일과 실행 방법을 요구하는가? |
| 8 | 검증 방법 | 테스트 입력과 예상 결과를 요구하는가? |
| 9 | 오류 점검 | 발생 가능한 오류와 주의점을 요구하는가? |
| 10 | 개선 방향 | 다음 단계에서 개선할 점을 요구하는가? |

## AI에게 프롬프트 생성을 요청하는 기본 형식

아래 형식을 사용하면 AI가 이 문서의 기준에 맞춰 코드 요청 프롬프트를 만들 수 있다.

```text
다음 개발 단계에 사용할 코드 요청 프롬프트를 작성해줘.

프로젝트:
C++ 기반 로컬 AI 채팅 프로그램

학습 대상:
C++ 초보 학생

현재 단계:
[예: 1단계 - 콘솔 입력과 종료 명령어 구현]

이번 단계의 목표:
[이번 단계에서 배우고 싶은 내용]

반드시 포함할 기능:
[이번 단계에서 구현할 기능 목록]

아직 제외할 기능:
[이번 단계에서 구현하지 않을 기능 목록]

프롬프트를 작성한 뒤, 이 문서의 "프롬프트 검토 항목"에 따라 스스로 검토하고 개선해줘.

답변 형식:
1. 프롬프트 초안
2. 항목별 자가 검토
3. 부족한 점
4. 개선 방향
5. 개선된 최종 프롬프트
```

## AI에게 기존 프롬프트 검토를 요청하는 기본 형식

이미 작성한 프롬프트가 있을 때는 아래 형식을 사용한다.

```text
아래 프롬프트 초안을 PROMPT_REVIEW.md의 기준에 따라 검토해줘.

검토할 때는 다음 형식으로 답변해줘.

1. 항목별 평가
2. 부족한 점
3. 수정 방향
4. 개선된 최종 프롬프트

검토 대상 프롬프트:
[여기에 검토할 프롬프트를 붙여넣기]
```

## 좋은 프롬프트의 최종 판단 기준

좋은 프롬프트는 다음 질문에 모두 답할 수 있어야 한다.

1. 학생이 이번 단계에서 무엇을 배울지 알 수 있는가?
2. AI가 불필요하게 어려운 코드를 만들 가능성을 줄였는가?
3. 결과물이 실행 가능하고 검증 가능한가?
4. 학생이 코드를 읽고 설명할 수 있도록 요구했는가?
5. 다음 단계로 자연스럽게 이어질 수 있는가?
