# 프롬프트 검토 기준

이 문서는 Codex가 프롬프트 초안을 검토할 때 사용하는 기준이다.
프로젝트 목표나 개발 계획을 설명하지 않고, 프롬프트 품질을 평가하는 데 필요한 항목만 다룬다.

## 검토 대상

다음과 같은 프롬프트를 검토한다.

1. 로컬 LLM 설치와 실행을 요청하는 프롬프트
2. C++ 로컬 AI 채팅 프로그램 코드 작성을 요청하는 프롬프트
3. 기존 코드의 오류 검증, 개선, 리팩터링을 요청하는 프롬프트
4. 단계별 학습용 개발 요청 프롬프트

## 핵심 검토 기준

프롬프트 초안이 아래 조건을 만족하는지 확인한다.

| 번호 | 검토 항목 | 확인 질문 |
| --- | --- | --- |
| 1 | 단계 명확성 | 현재 몇 단계의 작업인지 분명한가? |
| 2 | 목표 명확성 | 이번 요청에서 무엇을 얻고 싶은지 분명한가? |
| 3 | 범위 제한 | 한 번에 너무 많은 기능을 요청하지 않는가? |
| 4 | 제외 조건 | 아직 하지 않을 작업을 명확히 적었는가? |
| 5 | 수준 적합성 | C++ 초보자가 이해할 수 있는 수준을 요구하는가? |
| 6 | 실행 가능성 | 결과물을 실제로 실행하거나 확인할 수 있는가? |
| 7 | 검증 방법 | 테스트 방법, 확인 절차, 예상 결과를 요구하는가? |
| 8 | 설명 요구 | 코드 또는 절차의 실행 흐름 설명을 요구하는가? |
| 9 | 오류 점검 | 발생 가능한 오류와 확인할 항목을 요구하는가? |
| 10 | 다음 단계 | 다음에 개선하거나 확장할 방향을 요구하는가? |

## 로컬 LLM 관련 추가 검토 기준

로컬 LLM을 다루는 프롬프트라면 아래 항목도 확인한다.

1. 외부 클라우드 API가 아니라 로컬에서 실행되는 LLM을 기준으로 하는가?
2. 설치, 모델 실행, 응답 확인 중 어떤 작업을 요청하는지 분명한가?
3. 운영체제, 터미널, 실행 도구 등 필요한 환경 정보가 포함되어 있는가?
4. 너무 큰 모델, 복잡한 GPU 설정, 모델 튜닝처럼 초반 단계에 과한 작업을 제한했는가?
5. 나중에 C++ 프로그램과 연결할 방식이 확인 가능하도록 요청했는가?

## C++ 코드 요청 관련 추가 검토 기준

C++ 코드 작성을 요청하는 프롬프트라면 아래 항목도 확인한다.

1. 생성할 파일 이름이 분명한가?
2. 사용할 C++ 문법 수준이나 라이브러리 범위를 제한했는가?
3. 컴파일 방법과 실행 방법을 요구했는가?
4. 입력 예시와 예상 출력을 요구했는가?
5. 학생이 직접 설명할 수 있도록 주요 변수, 함수, 실행 흐름 설명을 요구했는가?

## Codex 답변 형식

프롬프트를 검토할 때 Codex는 다음 형식으로 답변한다.

1. 전체 평가
2. 항목별 검토
3. 부족한 점
4. 수정 방향
5. 개선된 최종 프롬프트
